# Feature Screenshots

Drop screenshots for each major release into a folder named after the release tag.

## Structure

```
screenshots/
  v80.0/
    feature-1.png    ← first featured screenshot (used for first email feature block)
    feature-2.png    ← second featured screenshot
  v81.0/
    feature-1.png
```

## Guidelines

- **Dimensions**: Aim for 1400×900px or similar widescreen ratio
- **Format**: PNG preferred, JPG accepted
- **Content**: Raw dashboard screenshots — the mockup generator will wrap them in the presentation frame automatically
- **Naming**: Any filename works; files are picked up alphabetically
- **Limit**: Up to 5 screenshots per release (one per feature block)

If no screenshots are provided for a release, the email is sent as text-only.
